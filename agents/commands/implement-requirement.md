# implement-requirement

## Instruction

1. Read `agents/contexts/instruction.md` first to understand the context and constraints.
2. Read `agents/contexts/implemented.md` to see the current progress.
3. Identify the requirement to implement from `agents/contexts/instruction.md` (e.g., the next `additional-requirement-*.md` file).
4. Read the detailed instruction in the corresponding `agents/contexts/additional-requirement-*.md` file.
5. Follow the breakdown steps in the file sequentially.
6. After finishing EACH step:
   - Check ESLint for errors.
   - If frontend code is changed, verify it works as expected.
   - Update `agents/contexts/implemented.md` to mark the step as completed.
7. If the requirement changes the workflow or adds a new constraint, update `agents/contexts/instruction.md` accordingly.
